During my internship and learning journey with **First Cloud AI Journey**, I participated in events related to cloud computing, generative artificial intelligence, Prompt Engineering, and practical AWS learning through hands-on experience.

These events helped me strengthen my technical knowledge and develop a more practical mindset toward learning and building applications on the cloud. Through the sharing sessions, I gained a clearer understanding of how AI can be applied to real-world systems, how Prompt Engineering improves the output quality of large language models, and how AWS can be learned effectively through labs, real projects, and safe experimentation tools.

---

## List of Events

### [4.1 - Event 1: FCAJ Community Day](4.1-Event1/)

This event introduced the importance of Prompt Engineering, the structure of an effective prompt, advanced prompting techniques, token economics, and the Proptimizer application built on AWS serverless architecture.

Through this event, I gained a better understanding of how to build structured prompts, how to optimize interactions with large language models, and how a real-world AI application can be deployed using AWS services such as CloudFront, S3, Cognito, API Gateway, Lambda, Bedrock, DynamoDB, and CloudWatch.

Date & Time: 09:00, May 9, 2026

Location: 26th Floor, Bitexco Financial Tower, 02 Hai Trieu Street, Saigon Ward, Ho Chi Minh City

Role: Attendee

### [4.2 - Event 2: FCAJ Community Day](4.2-Event2/)

This event focused on practical methods for learning AWS through hands-on experience. The session emphasized that effective cloud learning should not stop at reading documentation or watching tutorials, but should be combined with practice, project building, service experimentation, and learning from mistakes.

The main content of the event focused on **AWS Cloud Quest** and **Floci**. AWS Cloud Quest was introduced as a gamified learning platform that provides guided labs, realistic scenarios, and official AWS badges. This tool helps learners build cloud skills through a more structured and accessible learning path.

In addition, the event introduced **Floci**, an open-source local AWS simulator. Floci allows developers to develop, test, and experiment with AWS workloads on their local machines without directly deploying to the real cloud environment. This helps learners reduce the risk of unexpected cloud costs and test cloud ideas more safely.

Date & Time: 09:00, May 30, 2026

Location: 26th Floor, Bitexco Financial Tower, 02 Hai Trieu Street, Saigon Ward, Ho Chi Minh City

Role: Attendee

## Knowledge Gained

After participating in these events, I gained a better understanding of:

- The role of Prompt Engineering in improving the output quality of LLMs.
- How to build clear prompts using roles, context, instructions, constraints, and expected output formats.
- Advanced prompting techniques such as Chain-of-Thought, Self-Consistency, Tree-of-Thoughts, RAG, and Role Prompting.
- Token economics and the importance of writing concise, focused, and well-structured prompts.
- How a real-world AI product can be built on AWS using serverless architecture.
- Effective methods for learning AWS through hands-on practice, project building, service experimentation, and learning from mistakes.
- How AWS Cloud Quest supports AWS learning through guided labs, real-world scenarios, and official badges.
- The role of Floci in simulating AWS workloads locally, supporting testing, and reducing the risk of unexpected cloud costs.

---

## Personal Reflection

These events changed the way I approach learning and working with cloud and AI technologies. From the Prompt Engineering event, I learned that effective interaction with AI requires clear requirements, sufficient context, and a specific expected output format instead of short and unclear prompts.

In addition, the session about AWS Cloud Quest and Floci helped me realize that AWS is best learned by combining theory with hands-on practice. By working on labs, building small projects, testing services, and observing errors, I can better understand how cloud services behave in real situations.

In particular, the content about Floci gave me a new perspective on testing AWS workloads in a local environment before deploying them to the real cloud. This is useful for students and AWS beginners because it can reduce the risk of unexpected costs during learning and development.

Overall, both events directly supported my internship process, especially while developing the AWS BILLO project. I gained more knowledge about using AI effectively, learning AWS in a structured way, and paying closer attention to cost management when deploying applications on the cloud.
