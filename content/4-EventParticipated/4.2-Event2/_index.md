# Practical AWS Learning with AWS Cloud Quest and Floci

## Event Objectives

- Introduce practical methods for learning AWS through hands-on experience instead of relying only on theory.
- Share how **AWS Cloud Quest** helps learners build cloud skills through guided labs, real-world scenarios, and official AWS badges.
- Introduce **Floci** as an open-source local AWS simulator that supports developing and testing AWS workloads in a local environment.
- Help students and cloud beginners practice AWS more safely while reducing the risk of unexpected cloud costs.
- Encourage learners to approach AWS by building projects, experimenting with services, and learning from mistakes.

---

## Key Highlights

### 1. Learning AWS Through Hands-on Experience

One of the most important points of the event was the method of learning AWS through practice. The speaker emphasized that cloud learning becomes more effective when learners directly build projects, experiment with services, encounter errors, and learn how to solve them.

If learners only watch tutorials or read documentation, they may understand basic concepts but find it difficult to deeply understand how AWS services work in real situations. In contrast, hands-on practice allows learners to observe how resources are created, configured, connected, and operated.

This learning method helps learners:

- Understand how AWS services work more clearly.
- Remember knowledge better through practical experience.
- Learn how to troubleshoot errors when deploying systems.
- Develop the mindset of building and testing cloud applications.
- Become more confident when applying AWS to real projects.

---

### 2. AWS Cloud Quest

The event introduced **AWS Cloud Quest** as a gamified learning platform that helps learners approach AWS in a more visual and structured way.

AWS Cloud Quest provides guided labs, real-world scenarios, and official AWS badges. Instead of only learning from documents, learners can complete interactive missions, solve cloud-related problems, and gradually build practical AWS skills.

Some key features of AWS Cloud Quest include:

- An interactive learning environment.
- Clearly guided hands-on labs.
- Realistic cloud scenarios.
- A structured path for building AWS skills.
- Official AWS badges to recognize learning progress.

Through AWS Cloud Quest, learners can approach AWS in a more practical way. This is especially useful for students and beginners because it makes cloud services easier to access and understand.

---

### 3. Floci: Local AWS Simulator

In addition to AWS Cloud Quest, the speaker also introduced **Floci**, an open-source local AWS simulator.

Floci allows developers to develop, test, and experiment with AWS workloads on their local machines without immediately deploying resources to the real AWS cloud environment. This is useful for cloud beginners because they can test ideas before creating real AWS resources.

Floci provides several benefits:

- Supports local development and testing.
- Allows AWS workloads to be tested before cloud deployment.
- Reduces the risk of unexpected cloud costs.
- Helps learners become familiar with the behavior of AWS workloads.
- Creates a safer environment for experimentation and learning from mistakes.

For students, cost is an important factor when learning cloud computing. Therefore, using local simulation tools such as Floci can make the AWS learning process safer and easier to control.

---

### 4. Cost Awareness When Learning Cloud

Another important point from the session was cost control when learning AWS. Beginners may accidentally create unnecessary resources or forget to delete resources after testing, which can lead to unexpected billing.

Through this event, I understood that learning cloud is not only about knowing how to create services. It is also important to know how to control, monitor, and clean up resources. Tools such as AWS Cloud Quest and Floci can help learners practice more safely before deploying workloads to the real AWS environment.

Some important points include:

- Always check created resources after practice.
- Delete resources that are no longer used.
- Monitor costs when learning and testing on AWS.
- Prioritize guided labs or local simulation environments when starting out.
- Understand which services may generate costs before deployment.

## Lessons Learned and Personal Reflection

### Changing the Way I Learn AWS

This event changed the way I think about learning AWS. Previously, I thought that reading documentation and watching tutorials were enough to understand cloud knowledge. However, after this sharing session, I realized that cloud learning needs to be combined with regular hands-on practice.

Directly building projects, testing services, and solving errors helped me understand the nature of AWS services more clearly. This method is also suitable for my internship process because cloud knowledge becomes truly valuable when it can be applied to real systems.

### Practical View of AWS Learning Tools

The introduction to AWS Cloud Quest helped me learn about a more structured and accessible AWS learning tool. Its guided labs and real-world scenarios help learners avoid being overwhelmed when working with many different cloud services.

In addition, Floci gave me a new perspective on testing AWS workloads in a local environment. This is useful because learners do not always need to deploy directly to the real cloud, especially during the testing stage or when learning basic concepts.

### Better Awareness of Cloud Costs

Another important lesson from the event was the need to pay attention to costs when learning and developing on AWS. Cloud experimentation without proper control may lead to unexpected expenses.

Therefore, I need to build the habit of checking resources, monitoring costs, cleaning up the environment after practice, and considering local simulation tools before deploying to the real cloud environment.
